Taller arquitectura de software - GA4-220501095-AA2-EV06.

¿Qué entiende por arquitectura de software?

La arquitectura de software se refiere a la estructura organizativa y de alto nivel de un sistema de software. Es la representación de las principales decisiones de diseño que determinan cómo los componentes del software interactúan entre sí y con los sistemas externos. En esencia, es el esqueleto o el plano maestro de una aplicación o sistema de software que define la forma en que se construirá y funcionará.

¿Cuál es su función?

  • Organización: La arquitectura de software organiza los componentes y módulos del sistema, lo que facilita la gestión y el mantenimiento del código fuente.
  • Comunicación: Define cómo los diferentes componentes se comunicarán entre sí, lo que asegura que la información fluya de manera eficiente a través del sistema.
  • Escalabilidad: Permite que el sistema se adapte fácilmente a los cambios en los requisitos y la carga de trabajo. Una arquitectura bien diseñada facilita la escalabilidad.
  • Mantenimiento: Facilita la identificación y corrección de problemas, así como la incorporación de nuevas características o mejoras sin afectar negativamente al sistema existente.
  • Reutilización: Promueve la reutilización de componentes y módulos, lo que ahorra tiempo y recursos en el desarrollo de software.

¿Cómo se elabora la arquitectura de software?

La elaboración de la arquitectura de software implica varios pasos:

  • Requisitos: Comprender y documentar los requisitos funcionales y no funcionales del sistema. Esto proporciona la base para tomar decisiones arquitectónicas informadas.
  • Identificación de Componentes: Identificar los componentes principales del sistema y sus responsabilidades.
  • Interacción: Definir cómo los componentes interactuarán entre sí y con sistemas externos.
  • Patrones Arquitectónicos: Selección de patrones arquitectónicos adecuados (por ejemplo, MVC, microservicios, monolito, etc.) según los requisitos y las restricciones del sistema.
  • Diseño de Interfaces: Diseñar las interfaces entre componentes para garantizar la cohesión y el acoplamiento adecuados.

¿Cómo lograr una buena arquitectura?

Para lograr una buena arquitectura de software, hay que considerar lo siguiente:

  • Claridad y Simplicidad: Mantén la arquitectura lo más clara y simple posible. Evita la complejidad innecesaria.
  • Separación de Responsabilidades: Divide las responsabilidades de manera adecuada entre los componentes y sigue los principios SOLID y DRY (Don't Repeat Yourself).
  • Escalabilidad: Diseña para la escalabilidad, permitiendo que el sistema crezca con el tiempo.
  • Mantenibilidad: Haz que el código sea fácil de mantener y modificar. Utiliza buenas prácticas de codificación y documentación.
  • Seguridad: Considera las medidas de seguridad en todas las capas de la arquitectura.

¿Cuáles son los elementos de diseño de una arquitectura de software?

Los elementos clave en el diseño de una arquitectura de software incluyen:

  • Componentes: Los módulos o bloques de construcción del sistema.
  • Conexiones: Las formas en que los componentes se comunican entre sí.
  • Interfaces: Las reglas y protocolos que gobiernan la interacción entre componentes.
  • Patrones Arquitectónicos: Las estructuras de alto nivel que guían la organización del sistema, como el patrón MVC, el patrón de microservicios, etc.
  • Decisiones de Diseño: Las elecciones específicas sobre cómo se implementarán los componentes y las conexiones.
  • Restricciones y Requisitos No Funcionales: Las restricciones de rendimiento, seguridad, escalabilidad, etc., que deben tenerse en cuenta.
